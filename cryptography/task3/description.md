### **Цифровая подпись**

##### *Описание задачи*

В этой задаче мы реализуем механизм создания и проверки подписи с использованием HMAC (Hash-based Message Authentication Code) и общего секретного ключа. Хотя классические цифровые подписи используют асимметричную криптографию (пару из открытого и закрытого ключей), HMAC с общим секретом является отличным способом понять основные принципы на базе стандартных библиотек Python.

##### *Цель*

Реализовать две функции:

1. sign_message(message, secret_key) — создает цифровую подпись (HMAC) для сообщения.
2. verify_signature(message, signature, secret_key) — проверяет действительность подписи для сообщения.

Требования к функции sign_message:

* Принимает два параметра:
    * message (строка) — сообщение для подписи
    * secret_key (строка) — общий секретный ключ
* Использует алгоритм HMAC-SHA256 (из модулей hmac и hashlib) для вычисления подписи
* Возвращает подпись в виде строки шестнадцатеричных символов (hex digest)

Требования к функции verify_signature:

* Принимает три параметра:
    * message (строка) — сообщение
    * signature (строка) — полученная подпись (в hex формате)
    * secret_key (строка) — тот же секретный ключ, что использовался для подписи
* Пересчитывает HMAC-SHA256 для сообщения с использованием ключа
* Сравнивает пересчитанную подпись с предоставленной signature, используя безопасную функцию hmac.compare_digest()
* Возвращает True, если подписи совпадают (подпись действительна), и False в противном случае
