### **Защитная стена: анализ логов файрвола**

##### *Описание задачи*

Файрвол (межсетевой экран) — это первая линия обороны в сетевой безопасности. Он контролирует входящий и исходящий сетевой трафик на основе предопределенных правил безопасности, разрешая легитимные соединения и блокируя подозрительные или нежелательные.

Логи файрвола содержат бесценную информацию о том, какой трафик был разрешен, а какой — заблокирован. Анализ этих логов позволяет:

* Понять, какие правила файрвола срабатывают чаще всего.
* Обнаружить попытки несанкционированного доступа или сканирования сети.
* Выявить источники потенциальных угроз (IP-адреса, с которых идет вредоносный трафик).
* Оценить общую картину безопасности сети.

##### *Цель*

Написать скрипт на Python, который анализирует упрощенный лог-файл файрвола, выявляет записи о заблокированных соединениях и формирует отчет, включающий:

1. Общее количество заблокированных пакетов.
2. Топ-5 IP-адресов источников, с которых приходили заблокированные пакеты.
3. Топ-5 портов назначения, на которые пытались отправить заблокированные пакеты.

Формат лога, с которым мы будем работать (похож на UFW):
May 21 12:35:22 fw kernel: [UFW BLOCK] IN=eth1 OUT= MAC=... SRC=1.2.3.4 DST=192.168.1.20 LEN=... PROTO=UDP SPT=54321 DPT=137 ...

Пример строки из лога UFW (Uncomplicated Firewall):
May 21 12:35:22 fw kernel: [UFW BLOCK] IN=eth1 OUT= MAC=00:11:22:33:44:55:66:77:88:99:aa:bb:cc:dd SRC=1.2.3.4 DST=192.168.1.20 LEN=40 TOS=0x00 PREC=0x00 TTL=117 ID=12345 PROTO=UDP SPT=54321 DPT=137 LEN=20
              
Программа должна:

1. Считать данные лога из стандартного ввода.
2. Распарсить каждую строку, извлекая действие (например, "BLOCK", "REJECT"), IP-адрес источника (SRC) и порт назначения (DPT).
3. Подсчитать статистику по заблокированным пакетам.
4. Вывести результат в формате JSON.

Ожидаемый формат вывода:

{
  "total_blocked_packets": 9,
  "top_5_blocked_source_ips": {
    "1.2.3.4": 5,
    "5.6.7.8": 3,
    "9.10.11.12": 1
  },
  "top_5_blocked_destination_ports": {
    "22": 4,
    "137": 2,
    "23": 1,
    "445": 1,
    "8080": 1
  }
}
